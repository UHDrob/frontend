Describe the main steps in the domain name address resolution process.

  1. The resolution process starts at the user's computer. When the URL is requested, the browser will begin by seeing if it already has the IP address for the domain in its cache. if it does, it can jump to step 13
  2. if the browser doesn;t know the IP address for the requested site, it will delegate the task to the DNS resolver, a software agent that is part of the operating systems. The DNS resolver also keeps a cache of frequently requested domains.
  3. otherwise, it must ask for outside help, which in this case is a nearvy DNS server, a special server that processes DNS requests. This server keeps a more substancial cache of domain name/IP address pairs. If the requested domain is in its cache then the process jumps to step 11.
  4. If the local DNS server doesn't have the IP address for the domain in its cache, then it must ask other DNS servers for the answer. Thankfully, the domain system has a greate deal of redundancy built into it. This means that in general there are many servers that have the answers for any fiven DNS requests. This redundancy exists not only at the local level but at the global level as well.
  5. If the local DNS server cannot find the answer to the request from an alternate DNS server, then it must get it from the appropriate top-level domain (TLD) name server. Our lcoal DNS server might already have a list of the addresses of the appropriate TLD name servers in its cache. In such a case, the process can jump to step7.
  6. if the local DNS server does not already know the address of the requested TLD server, then it must ask a root name server for that information. The DNS toot name servers astore the address of TLD name servers. IANA authorized 13 root servers, so all root requests will fo to one of those 13 roots. In practice, these 13 machines are mirrored and distributed around the world. There are over 500 root server machines.
  7. After receiving the address of the TLD name server for the requested domain, the local DNS server can now ask the TLD name server for the address of the requested domain. As part of the domain registration process, the address of the domain's DNS servers are sent to the TLD name servers, so this is the information that is returned to the local DNS server in step 8
  8. The user's local DNS server can now ask the DNS server for the requested domain. It should receive the correct IP address of the web server for that domain. This address will be stored in its own cache so that future requests for this domain will be speedier. That IP address can finally be returned to the DNS resolver in the requesting computer
  9. The browser will eventually receive the correct IP address for the requested domain. If the local DNS server were unable to find the IP address, it woudl return a failed response, which in turn would cause the browser to display an error message.
  10. Now that it knows the desired IP adddress, the browser can finally send out the request to the web server, which result in the web server responding with the requested resource.

Describe the four layesr in the four-layer network model

  • Application Layer:
    • Higher protocols that allow applications to interact with transport layer
    • HTTP, FTP, POP, etx
  • Transport Layer
    • Ensures transmissions arrive in order and without error
    • TCP, UDP
  • Internet Layer
    • Establishes connection, routing, and addressing
    • IPv4, IPv6
  • Link Layer
    • Responsible for physical transmission of raw bits
    • MAC

What is the Internet Protocol (IP)? Why is it important for web developers?

The Internet uses the Internet Protocol (IP) addresses, which are numeric codes that uniquely destinations on the Internet.

The details of the IP addresses can be important to a web developer. These are ocassions when one needs to track, record, and compare the IP address of a given web request.

How many distinct domains can be hosted at a single IP address?

The host header was introduced in HTTP 1.1. and it allows multiple websites to be hosted from the same IP address. Since requests for different domains can arrive at the same IP, the host header tells the server which domain at this IP address we are interested in.
